Zweistufiges Token-System. Primitive Werte → Semantic Tokens → Component Tokens. Tenants überschreiben nur --brand-* — alles andere leitet automatisch ab.
Brand (Tenant-overridable)
/* In tenants/augsburg.css */ :root { --brand-primary: #e05a00; --brand-primary-100: #fde8d4; --brand-primary-rgb: 224, 90, 0; }
Semantic Action (derived)

Components never reference --brand-* directly. They use --color-action-*, which derives from brand tokens. One tenant override → entire UI updates.

Scroll Reveal
Klasse .sr auf jedes Element. JS-IntersectionObserver fügt .v hinzu. Für gestaffeltes Timing: Parent als .sr-group, Children erben Delays.
Item 1 (delay 50ms)
Item 2 (delay 100ms)
Item 3 (delay 160ms)
<!-- HTML --> <div class="sr-group"> <div class="c-card sr">Item 1</div> <div class="c-card sr">Item 2</div> </div> /* JS (einmalig pro Seite) */ document.querySelectorAll('.sr').forEach(el => { new IntersectionObserver(([e]) => { if (e.isIntersecting) el.classList.add('v'); }, { threshold: 0.15 }).observe(el); });
Multi-Tenant System
Jede Bibliothek bekommt eine kurze CSS-Datei. Nur Brand-Farbe + Fonts. Der Rest des Systems leitet automatisch ab.
